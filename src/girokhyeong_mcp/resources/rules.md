# 기록형 풀이 룰 정본 (R1~R10 + 논증·어법 규칙)

> 단일 정본. 법학볼트의 `기록형풀이_룰_2026-05-05`·`독립판 서면작성 프롬프트 v3`·`판례탐색_전략`·`기록형_논리포섭검토`에 분산돼 있던 규칙을 한곳에 통합하고, 검증 도구를 `law_api`(법제처 직접 API)로 재배선했다(은퇴한 `korean-law-mcp` 참조 제거).

적용 범위: 변시 기록형, 가인 예선 참고서면, AI 대회 사건기록, 모의기록.

---

## R1. 기록 한정 (Record Grounding)
사실관계는 제공된 사건기록에 적힌 것만 사용한다. 새로운 사실 가정·기록 임의 보강 금지. 채점 핵심 = "기록에 있는 사실의 누락 없는 활용". 기록 밖 사실 창작 = 즉시 감점. 모든 사실 주장 뒤에 출처를 괄호로 표기한다 — `(사실관계 제N항)`·`[《문서명》 p.X | "발췌"]`. 문제의 [사실관계]와 당사자 일방의 상담·주장은 구분한다(일방 주장은 "~라고 주장한다"로만).

## R2. 미확인 보류 (Not-in-Record)
기록에서 확인 불가한 단정은 출력하지 않는다 → "기록에서 확인할 수 없습니다" 또는 `[자료 부족—보류]`. **보류 > 잘못된 단정.** 요건 매트릭스에서 사실이 없으면 칸은 `공백`으로 두고 지어내 채우지 않는다(no silent supplement).

## R3. 인용 의무 (Citation Mandatory)
- 사실 단정: `[《문서명》 p.X | "정확한 발췌"]` (기록 쪽수·증거번호 병기). 쪽수 없는 별첨·항목 구조는 `[《별첨N 항목》]`·`(사실관계 제N항)`·`(별첨2 갑구 순위 N번)`을 정식 인용으로 인정.
- 법리 단정: 조문 원문 또는 판례 사건번호 + 판시 발췌.
- 위치 식별자 없으면 `(확인 불가—위치 식별자 부족)`.
- **검증완료 판시만** 본문에 정식 인용형식 + 따옴표 발췌로 직접 인용·포섭. `[검증필요]` 후보 사건번호는 본문 노출 금지(정리표·메모에만). 본문은 "판례의 취지"로만 서술.
- **판시-사건 1:1 귀속**: 한 사건번호에 이질 판시 여러 개 묶음 금지.

## R4. 실재 판례·조문만 (No Fabricated Authority)
미확인 판례·조문·논문 인용 금지. **검증 순서**: ① `verify_text(서면)`로 인용 조문·사건번호 일괄검증(개별 `verify_case`·`verify_article`) → ② 안 되면 law.go.kr/glaw 수동 확인 → ③ 그래도 불가 시 `[검증필요]`. 인용마다 신뢰등급 태그: 직접 조회 확인 = `[확인]`, 미검증·기억의존 = `[검증필요]`.
- 검증필요 명제 위 소결 단정 금지(조건부로 낮춤). 도구 반환 로그 없이 "확인됨" 단정 금지 → `[부분검증: 사건명만 일치, 판시 미확인]`·`[검증필요: 후보 없음]` 2단 표기.
- **서브에이전트 판례는 메인이 1차 출처로 재대조 후에만 본문 인용**(서브 '확인' 보고만으로 금지).

## R5. 원문 우선 (Original-Text Priority)
조문·판례 원문을 우선한다. 판례 태도는 판결문 원문에 확인된 표현만 쓰고(전사·요약 표현 금지) 사건번호를 병기한다. 직접 인용("…")은 원문의 **연속된 부분문자열만**. 풀어 쓸 때도 핵심 표지어를 유지하고 충분/필요조건을 뒤집지 않는다("저당권의 목적되는 토지", "패소한 때에는 그 소가 제기된 때로부터 악의의 점유자로 본다"). 원문 없으면 `(원문 미포함)`.

## R6. 순환 인용 금지 (No Self-Citation)
직전 응답·이전 단계에서 모델 자신이 만든 요약·표·합성 문장을 다음 단계 출처로 재인용하지 않는다. 출처는 항상 원 기록 또는 공식 원문(법령정보센터·대법원·헌재·KCI). 1단계 정리표를 3단계에서 인용해도 표기는 원 기록 문서를 지시한다.

## R7. 한자→한글 (Korean Output)
甲→갑, 乙→을, 條→조. 외국어 법률용어는 한국어 대역 우선(+필요시 괄호). §·①②③ 기호는 유지. 본문에 라틴어·독일어·불필요한 한자 금지.

## R8. IRAC 변형 골격 (기록형 특화)
쟁점 → 법리(조문·판례) → 포섭(기록 발췌 인용) → 소결. **포섭이 핵심**: 포섭 문장마다 기록 인용("이 사건의 경우 ~므로"). 민사 = 요건사실 순서(권리근거→항변→재항변, 2열 표 원고|피고). 형사 변호인 = 공소사실 요지→쟁점별 의견→정상관계→결론. 대회가 원·피 양측 서면을 요구하면 한쪽만 지정돼도 양측 모두 작성(추첨 전 제출). **IRAC 라벨은 내부 구조로만, 본문에 노출 금지**(변호사 의견서 산문). 작성 후 논리·포섭 5축으로 자가점검.

## R9. 상대방 주장 예상·반박 (Adversarial Completeness)
상대방 청구원인·항변·재항변 최소 1개를 예상하고 반박을 서면 내부에 배치한다. 불리한 쟁점 회피 금지 — 처리방침을 명시 선택한다: (가) press(정면 반박: 사실차이·후행 대법원 판례) / (나) concede·pivot(인정 후 더 강한 쟁점 이동) / (다) drop(쟁점 취사). 선택 이유 1줄. 불리 판례 구별(distinguishing)은 ①사실관계 차이 ②후행 대법원 판례 순.

## R10. 추론 비노출·서면 문체
내부 추론은 출력하지 않는다. 최종 서면은 법원 제출 문체("~입니다/~하였습니다"). "조사해보니"·"~인 것 같다"·"생각된다" 등 메모·추측 문체 금지. 무주어 술어 금지(주어-술어 호응 확인). 대비 구문은 정렬("위 사안은 ~인 반면, 이 사건은 ~").

## 논증 구조 보강 (독립판 §3)
- **다중 안전망**: 핵심 쟁점은 주위적 구성이 무너져도 결론이 유지되도록 층화(주위적→예비적→최후). 각 층 독립 성립.
- **상대 최강 논거 먼저** 다룬다.
- **요건 매치업 매트릭스 + 갭식별**: 청구원인·항변마다 요건을 ①②③로 분해, `| 요건 | 충족 사실(출처+원문 발췌) | 반대·불리 사실 | 충족도 | 상태 |`. 상태 = 충족/일부/다툼/공백/불명. 공백·불명 행만 모아 갭 목록 + 입장별 의미(우리 요건이면 입증 보완·예비 재구성, 상대 요건이면 반박 공격점).

## 채점 설계 (가인 예선 배점)
쟁점포착 30 · 법리·판례조사 30 · 상대방논리대응 20 · 완성도 10 · 형식·인용 10.
- 쟁점 취사(30): 승소에 필요한 쟁점만. 불필요·불리 쟁점은 회피 아닌 최소 방어. 기계적 전부 나열 금지.
- 포섭 강제(30): 모든 법리에 사실관계 포섭 문장 1개 이상.
- 1:1 반박(20): 상대 최강 논거를 독립 항으로 정면 반박.

## 인용 표기 표준
- 판례: `대법원 YYYY. M. DD. 선고 NNNN다XXXXX 판결` / 약식 `대판 YYYY.M.DD, NNNN다XXXXX`. 전원합의체·결정은 그대로 병기(`대법원 … 전원합의체 판결`·`… 결정`).
- 헌재: `헌법재판소 YYYY. M. DD. 선고 YYYY헌마XXX 결정`.
- 조문: `「형법」 제257조 제1항`(정식) / `형법 §257①`(노트형).
- 기록: `[《피의자신문조서(이정한)》 p.7 | "발췌"]` — 문서명에 작성자·차수 병기.

## 검증 3단 (판례탐색 §5)
- `검증완료`: 번호·선고일·판시취지 1차소스/`verify_case` 원문 대조 일치 → 본문 직접인용 승격.
- `부분검증`: 번호·선고일 일부만, 또는 취지만 확인 → `[부분검증: 사건명만 일치, 판시 미확인]`.
- `검증필요`: 확인 불가 → `[검증필요: 후보 없음]`, 본문 노출 금지.

## 신뢰·갭 마커 (연구 N3·N5·N7)
`[확인]` / `[검증필요]` / `[UNCERTAIN: 사유]` / `[GAP — 사실 추가확인]` / `[CITE NEEDED]`. silent gap 금지(빈칸은 의도적 표시). 핀사이트가 명제 일부만 지지하면 분할(misgrounded 표시). 약한 논증은 솔직히 — 누르기/양보/포기 옵션 명시.
